# 私有化多模态大模型平台执行顺序与整体流程

## 阶段 0：目标与范围确认

1. 确认使用范围：  
   - 用户：内部几十人  
   - 场景：复杂文档/PDF/多图分析 + 日常问答 + Agent 自动化  
2. 确认安全要求：  
   - 必须：离线 / 内网部署  
   - 是否允许访问公司内网其他系统（工单、报表、数据库）  
3. 确认预算与周期：  
   - 决定是“一台强服务器”还是“小集群”  

> 产出：一页纸需求说明 + 预算范围。

---

## 阶段 1：硬件采购与机房准备

1. 按采购单走招标 / 采购流程  
2. 机房/机柜准备：  
   - 机柜空间、供电（GPU 服务器耗电高）、网线（只接内网）  
3. 到货后运维完成：  
   - 上架、接电、接内网  
   - 安装基础 OS（建议：Ubuntu Server LTS）  

> 产出：一台（或多台）装好系统、能通过内网 SSH 登陆的 GPU 服务器。

---

## 阶段 2：基础软件环境搭建

在 GPU 服务器上完成：

1. 系统与驱动  
   - 配置软件源  
   - 安装 NVIDIA 驱动  
   - 安装 CUDA（如不完全使用容器）  

2. 容器与目录结构（推荐 Docker）  

   示例目录：

   ```bash
   /opt/ai-platform/
     ├── models/        # 模型权重
     ├── data/          # PDF、文档等原始数据
     ├── vectorstore/   # 向量库数据
     ├── services/      # 各服务的配置
     └── logs/          # 日志
   ```

3. 基础软件栈  
   - Python 3.10+  
   - Docker + nvidia-container-toolkit  
   - Git 等工具  

> 产出：可跑容器的 GPU 环境。

---

## 阶段 3：模型选型与下载（离线准备）

在一台可上网的机器上：

1. 确定模型组合：  
   - 多模态主力模型（14B~32B 中文多模态）  
   - 轻量多模态模型（7B 左右）  
   - 纯文本/路由模型（轻量聊天/指令模型）  
   - Embedding 模型（中文向量检索）

2. 从官方仓库下载模型，打包：  
   - 使用 git-lfs 或 huggingface_hub  
   - 压缩为 tar.gz 等格式  

3. 拷贝到离线服务器并解压到：`/opt/ai-platform/models/`

> 产出：服务器本地有完整模型文件，无需联网推理。

---

## 阶段 4：使用 vLLM 部署模型服务

1. 在服务器上部署 vLLM：  
   - 使用 Docker 镜像：`vllm/vllm-openai` 或
   - 使用 pip 本地安装 vLLM 包  

2. 为每个模型启动独立 vLLM 服务（端口示例）：  
   - 多模态大模型：8001  
   - 轻量多模态：8002  
   - 文本/路由模型：8003  
   - （可选）代码模型：8004  

3. 对外统一使用 OpenAI API 格式：  
   - `/v1/chat/completions`  
   - `/v1/embeddings`  

4. 内部测试：  
   - curl / Postman / Python 脚本  
   - 调整 `max_model_len`、`max_num_seqs` 等参数  

> 产出：OpenAI 兼容的模型网关层。

---

## 阶段 5：文档/PDF 处理与 RAG 向量检索

1. 选择向量库（Qdrant / Milvus / Chroma）并部署  
2. 搭建文件解析服务：  
   - PDF 文本抽取、表格抽取、图片截取/OCR 等  
3. 文档切分与向量化：  
   - 调用本地 embedding 模型  
   - 将向量 + metadata 写入向量库  
4. 暴露 RAG 查询 API：  
   - 输入：自然语言问题（+ 文档范围）  
   - 输出：相关文本片段 + 文档位置信息  

> 产出：供 Agent 调用的“文档大脑”。

---

## 阶段 6：Agent 编排层搭建

1. 选用框架（LangChain / LlamaIndex 等）或自建  
2. 封装工具（Tools）：  
   - 文档检索：`search_docs`  
   - 文件解析：`parse_pdf`  
   - 业务 API：`run_sql`、`get_ticket_status` 等  
3. 设计多个 Agent 角色：  
   - 文档分析 Agent  
   - 知识问答 Agent  
   - 运维/报表 Agent 等  
4. 对外统一接口：`/api/chat` 或类似入口  

> 产出：具备工具调用能力的智能 Agent 层。

---

## 阶段 7：前端与内网集成

1. Web 前端：  
   - 通用聊天界面  
   - 文档分析界面（上传/选择文档 + 高亮引用）  
2. 权限控制：  
   - 接入 SSO/LDAP/AD  
   - RAG 层按用户/部门过滤文档  
3. 内网系统集成：  
   - 工单系统/报表系统通过 API 调用 Agent  

> 产出：所有人可通过内网访问的统一 AI 门户。

---

## 阶段 8：监控与治理

1. 监控：  
   - GPU 利用率、延迟、QPS、错误率  
2. 日志与反馈：  
   - 匿名化问题与答案（视合规要求）  
   - 梳理失败案例优化 RAG/提示词/模型  
3. 安全与合规：  
   - 敏感信息策略  
   - 内部数据使用说明与权限策略
